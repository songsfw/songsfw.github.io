弹窗

弹窗为jquery编写的插件，基本包含了所有的弹窗场景，包括 居中弹窗，右划弹窗，下划出弹窗，3秒提示层

算是修改了从易车惠开始的很久以来的一个bug。

可关闭调用方法

所有需要关闭的弹层都由此调用，参数后边详解

约定的html结构

直接在弹窗元素上调用弹窗插件，插件只有一个很简单的功能：显示弹窗，所有需要关闭的弹窗的动作/形式均由css控制，可以扩展更多弹窗形式而只需要添加css样式即可，不必修改js，做到了很好的样式和行为分离

pop-panel为弹层默认class(必须),绑定了弹窗默认样式，right-panel为从右往左划出，其他还有

btm-panel : 下往上

center-panel : 居中弹出

pop-con : 为弹窗内容区，加入了overflow:auto;-webkit-overflow-scrolling:touch; 弹窗内容溢出滚动

jsClose : 默认为关闭事件class,关闭当前活动弹窗，关闭遮罩层(mask)

所有弹窗level参数默认为1，即1级弹窗，当我们需要2级弹窗时，使用level:2,然后本弹窗中的jsClose关闭事件自动变为只关闭本2级弹窗

PS : 所有以上弹窗除提供默认的jsClose关闭事件外，也都可以调用插件的关闭方法来关闭弹窗，以此来满足一些回调方法，比如执行完指定方法后关闭弹窗

$.fn.popPanel.hide($(".quiz-page"),$(".mask"))

第一个参数为要关闭的弹窗，第二个参数为遮罩层

3秒隐藏提示层

$(".tip-pop").popPanel({type:2,desc:"暂无活动"})

所有可关闭的弹窗 type 默认为1，渐隐提示层 type 为2，第二个参数desc 为提示文字，type值为2 的时候才生效

PS ： 弹窗提供了一个回调函数，以备不时之需

addClass3 ： 为css3属性添加兼容

toucheContent : 使用js模拟滚动（scroll）

motion-btn : 模拟android按钮的水波纹交互效果

其他一些东西

还是感觉自己技术有限吧。很多方法都加了很多参数来区别各种情况，总感觉这不会是最优的解决方案。肯定是有更好的写法

比如 getLocalAct(actMap,filter) 调用线下活动接口，actMap 为 传的ajax参数，filter为是否在进行筛选，团购也是如此，因为进行筛选的时候是不需要全国数据补齐的。需要区分

setACtData(data,n) 插入线下活动，data为接口拿到的数据，n为是否在进行全国数据补齐

这里要说一下非常扯的地方，接口过来的数据是分页好的，而产品要求移动端首页要显示当前区域活动，如果当前区域活动数量过少，用全国数据补齐一页。因此 线下活动 基本上每次显示都会请求2次，而用全国数据补齐的时候，还要去重之前已经显示过的活动

线下活动和团购逻辑是一样的，方法基本上也是一样的，但我就是懒得将方法写成可复用的了…..
