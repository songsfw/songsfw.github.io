作用域 作用域链

js代码执行过程

编译器 –> 编译 –> 词法语法分析，生成可执行代码，确定作用域

引擎 –> 执行 –> 执行上下文创建，代码执行，垃圾回收

执行上下文生命周期

创建： 生成变量对象，建立作用域链，确定this指向

执行：变量赋值，函数引用，执行其他代码

执行完毕后出栈，等待回收

大概就是这样一个执行过程，从以上过程其实就可以看出 作用域和this指向 是在什么阶段确定的了

作用域

函数作用域 function(){}

函数作用域是指 属于这个函数的所有变量都可以在整个函数的范围内使用及复用

  • 每个函数都会为自身创建一个作用域

  • 函数表达式可以匿名，函数声明必须是具名的

  • 立即执行函数表达式(IIFE) (function(){})() 有两个优势，函数名不会污染所在作用域，并且能够自动执行(无需调用)

  • IIFE 有一个普遍的进阶用法是把它们当做函数调用并传参进去

  • 什么时候建议使用？当一个函数仅执行一次的时候

立即执行函数还有几个有趣的特点 下次详说

举例

  • JavaScript中的函数运行在它们被定义的作用域里,而不是它们被执行的作用域里(函数定义时确定作用域，执行时确定this指向)

块作用域

变量的声明应该距离使用的地方越近越好，并最大限度的本地化，块作用域是用来对之前的最小授权原则进行扩展的工具，将代码从在函数中隐藏信息扩展为在块中隐藏

以上是一个伪块作用域，我们希望变量i只在for循环中使用，但是此处的i实际上是在全局作用域

非常少的人能注意到 js中的try/catch 的catch分句会创建一个块作用域，其中声明的变量只在catch中有效

举例

但是es6中出现了更方便的 创建块作用域的方法,即新的变量声明方法 let

let除了可以创建块作用域，还有一个区别于var的点，let不会对声明的变量进行提升

  • 块作用域另一个非常有用的原因和闭包及垃圾回收的机制相关

除了let 还有const也可以创建块作用域，const 值为固定值(常量)

彻底搞懂闭包

1 闭包与作用域链息息相关；

2 闭包是在函数执行过程中被确认。

闭包的定义是 函数可以记住并访问所在的作用域时，就产生了闭包，即使这个函数在当前作用域外执行

  • 简单来说，假设函数b在函数a的内部进行定义了，并且当函数b在执行时，访问了函数a内部的变量对象，那么a就是一个闭包。

闭包原理

每个函数创建执行时会产生一个可执行上下文，函数执行完毕 可执行上下文销毁

可执行上下文类似但不等同于作用域

如上 function a 的执行上下文为 AOa -> GO(全局上下文)
function b 的执行上下文为 AOb -> AOa -> GO

在函数a外部我们无法访问变量aa，函数b可以访问到

当函数b return到外部时，函数a执行完毕 执行上下文销毁，但是b函数此时还保持着对AOa的引用，因此依旧可以访问到变量aa

无论使用何种方式对函数类型的值进行传递，当函数在别处被调用时都可以观察到闭包。

如果将函数（访问它们各自的词法作用域）当作第一级的值类型并到处传递，
你就会看到闭包在这些函数中的应用。
在定时器、事件监听器、Ajax 请求、跨窗口通信、Web Workers 或者任何其他的异步（或者同步）任务中，
只要使用了回调函数，实际上就是在使用闭包！
