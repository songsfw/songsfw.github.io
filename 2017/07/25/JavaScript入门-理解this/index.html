初学者很容易被this的指向搞得晕头转向，但是知道了原理之后，会很快理解this的运行机制

大概30分钟，彻底理解this

this的作用:使得代码更简洁 可复用，用一种更优雅的方式实现对象的引用

首先要记住的一点，this的指向是在函数执行的时候确定的，而不是函数声明的时候，this最终指向调用他(函数)的对象

this的取值取决于他所在的函数如何被调用，下面是this可以获得新值的6种方式

  • this在全局范围内

    当this在任何函数外面被调用时，也就是说在全局环境中被调用 他默认指向window对象

  • this在普通函数内

    在简单函数中 this指向window. 即使你在对象的方法中调用一个简单函数, 在这个简单函数里面的this也是指向Window.

  • this在对象的构造函数内

    当使用new关键字创建一个对象的实例时，this指向这个实例

  • this在对象的方法内

    对象方法内的this指向对象本身

  • this在箭头函数内

  • this在一个时间侦听器内

new发生了什么？这里不展开说(够另起一篇了)，简单来说new关键字会创建一个空对象，然后调用call，将this指向这个空对象

判断this

现在我们可以根据优先级来判断函数在某个调用位置应用的是哪条规则。可以按照下面的顺序来进行判断：

  1. 函数是否在new 中调用（new 绑定）？如果是的话this 绑定的是新创建的对象。

var bar = new foo()

  1. 函数是否通过call、apply（显式绑定）或者硬绑定调用？如果是的话，this 绑定的是
    指定的对象。

var bar = foo.call(obj2)

  1. 函数是否在某个上下文对象中调用（隐式绑定）？如果是的话，this 绑定的是那个上
    下文对象。

var bar = obj1.foo()

  1. 如果都不是的话，使用默认绑定。如果在严格模式下，就绑定到undefined，否则绑定到
    全局对象。

var bar = foo()
